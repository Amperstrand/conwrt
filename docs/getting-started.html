Getting Started with conwrt

From zero to OpenWrt in 30 minutes.

0Prerequisites

Before you begin, ensure you have the following:

  • macOS (uses say for voice guidance, ifconfig for interface detection) or Linux
  • Python 3.10+
  • tcpdump and sudo for pcap capture and network monitoring
  • An ethernet cable and a router to flash
  • A firmware image OR willingness to request one from ASU

1Install conwrt

Clone the repository and set up a Python virtual environment:

git clone https://github.com/amperstrand/conwrt.git
cd conwrt
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e '.[dev]'

Verify the installation:

python3 scripts/conwrt.py --version

2Check supported devices

Run this command to see which device models conwrt supports:

python3 scripts/conwrt.py list

If your device is listed, you are in Stage 2 (automated flashing). If not listed, you need Stage 1 (AI-assisted discovery) — see the prompts/ directory for templates.

3Connect your router

Physically connect your computer to the router:

  • Plug an ethernet cable from your computer to one of the router's LAN ports (do NOT use WAN)
  • Ensure your computer's ethernet interface gets a DHCP lease from the router, or set a static IP in the router's subnet
  • Verify basic connectivity with ping <router-ip> or proceed to the next step and let conwrt discover the device

4Detect and identify the device

conwrt provides three commands to discover connected devices:

Scan the ethernet interface for connected devices:

python3 scripts/conwrt.py probe --json

Auto-detect the router and its boot state (off/uboot/openwrt):

python3 scripts/conwrt.py auto --no-menu

If the router is already running OpenWrt, fingerprint it for full identity:

python3 scripts/conwrt.py fingerprint <ip> --json

Explanation: probe finds devices on the interface. auto detects boot state and identifies the model. fingerprint extracts board_name, MAC address, kernel version, and installed packages via SSH (only works if OpenWrt is already running).

5Get firmware

You have two options for obtaining firmware:

Option A — Use an existing image:

  • Download a factory or sysupgrade image from the OpenWrt Firmware Selector or the ASU server
  • Place the .bin file somewhere accessible on your computer

Option B — Request a custom ASU image (recommended):

# Dry-run: see what would be built
python3 scripts/firmware-manager.py request --profile dlink_covr-x1860-a1 --dry-run

# Real request (bakes in your SSH key, password, use cases)
python3 scripts/conwrt.py flash --model-id dlink-covr-x1860-a1 --request-image --ssh-key ~/.ssh/id_ed25519.pub --wan-ssh

Custom ASU images automatically embed your SSH keys, set your preferred password mode, and pre-configure use cases (tethering, SQM, VPN, etc.) defined in config.toml.

6Flash

DRY RUN FIRST — detect recovery mode without uploading anything:

python3 scripts/conwrt.py flash --model-id dlink-covr-x1860-a1 --image firmware.bin --no-upload

Real flash — once dry-run succeeds:

python3 scripts/conwrt.py flash --model-id dlink-covr-x1860-a1 --image firmware.bin
SAFETY NOTE: Never use sysupgrade -F or any force flag. If sysupgrade rejects an image with a validation error, the image is wrong for the hardware. Do not force it. The -F flag bypasses hardware validation checks and can brick devices. See AGENTS.md for safety rules.

7Verify

After flashing, conwrt automatically verifies the flash and collects inventory:

  • Polls for SSH connectivity on the new device
  • Verifies the firmware version via /etc/openwrt_release
  • Collects device inventory (MAC address, serial number, installed packages) and appends to data/inventory.jsonl
  • Reports the randomly generated root password (if --no-password was not used)

The verification process ensures the flash succeeded and the device is running the expected firmware.

8Configure (optional)

After flashing, you can apply configuration defined in config.toml to a running router:

Preview what config.toml would apply (dry-run):

python3 scripts/conwrt.py profile plan --model-id dlink-covr-x1860-a1

Apply configuration to a running router:

# Dry-run first (no changes)
python3 scripts/conwrt.py configure --dry-run --ip 192.168.1.1

# Apply for real
python3 scripts/conwrt.py configure --ip 192.168.1.1

This step applies use cases, WiFi settings, network configuration, and other options from your operator profile.
